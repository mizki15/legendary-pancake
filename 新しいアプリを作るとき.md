アプリを新しく追加する手順は、今回行った**「Blueprint（ブループリント）方式」**に則ると、非常にシンプルで型が決まった作業になります。

新しいアプリ（例として `weather` アプリ）を追加する場合を想定して、ステップを解説します。

---

## 1. 新しいファイルを作成する (`weather.py`)

まず、新しい機能を記述するファイルを作ります。

```python
from flask import Blueprint, render_template

# 1. Blueprintを定義（名前 'weather' は他と被らないように）
weather_bp = Blueprint('weather', __name__)

# 2. ルート（URL）を書く
@weather_bp.route("/weather")
def index():
    # weather.html を templates フォルダに用意しておく
    return render_template("weather.html")

@weather_bp.route("/weather/detail")
def detail():
    return "天気情報の詳細ページです"

```

---

## 2. メインの `app.py` に登録する

作成したファイルを `app.py` に認識させます。

```python
# app.py の上部でインポート
from weather import weather_bp

# ...中略...

# app.register_blueprint が並んでいる場所に追記
app.register_blueprint(weather_bp)

```

---

## 3. HTMLテンプレートを作成する

`templates/weather.html` を作成します。
これで、ブラウザで `http://localhost:5000/weather` にアクセスすれば新しいアプリが表示されます。

---

## 4. (重要) リンクの書き方に注意

Blueprintを使うと、HTML内でのリンクの書き方が少し変わります。`url_for` を使っている場合は、**`Blueprint名.関数名`** という形式にします。

* **修正前:** `url_for('index')`
* **修正後:** `url_for('weather.index')`

---

## まとめ：アプリ追加のチェックリスト

| ステップ | 内容 |
| --- | --- |
| **① 新規ファイル** | `Blueprint` を定義し、`@名前_bp.route` で関数を書く |
| **② インポート** | `app.py` の冒頭で `from ファイル名 import 名前_bp` |
| **③ 登録** | `app.py` 内で `app.register_blueprint(名前_bp)` |
| **④ HTML** | `templates/` フォルダに必要なHTMLを設置 |
